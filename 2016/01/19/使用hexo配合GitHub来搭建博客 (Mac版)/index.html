使用hexo配合GitHub来搭建博客 (Mac版)

1. 安装配置环境

  1. 下载并安装Node https://nodejs.org/en/download/   ———>  (用于生产静态网页)
  2.  安装Git  

              注: 安装了Xcode的, 就会连带安装了Git,可以不安装Git , 如果你没有就在安装一个Xcode反正以后会用到的😊
              安装Git是为了把本地的hexo内容提交到github上去.
    3. GitHub账号
              用来做博客的远程创库, 域名, 服务器 (需要与本地的hexo建立链接),最好配置一下SSH Keys, 省的以后每次  提交博客的时候都要手动输入账号密码.
    4. 配置SSH Keys(可选项)
                 配置了SSH Keys ,以后每次登入自己的博客或自己的博客有改动提交的时候就不需要手动输入账号密码
                 官方配置教程:     
https://help.github.com/articles/generating-ssh-keys/

2. 安装Hexo

        1.  安装好Node 和 Git以后, 在编译器里输入以下指令来安装hexo
sudo npm install -g hexo
        2.  安装好hexo后, 对其进行初始化
                    1. 创建一个名为hexo的文件夹
mkdir hexo
        3. 进入刚创建的文件夹, 然后初始化, 代码如下
cd hexo
hexo init
npm install
        至此安装工作全部完毕

3. 生成静态页面

            1. 进入(cd)到创建的hexo文件目录下, 执行如下命令
hexo generate
//或者hexo g

4. 启动本地服务(可以浏览文章), 执行下面命令

hexo server
//或 hexo s
完成以后你可以通过这个进入看看http://0.0.0.0:4000/暂时性记忆

5.HTTPS URLs 和 SSH URLs的区别

          https是在任何情况下都能用的, ssh是要配置好ssh-key后方能使用

6. hexo的具体功能

  • 撰写博文
/** 进入终端 **/
cd hexo
/** 创建一个名为blogName的.m文件 **/
hexo new "blogName"   
生成的.md文件的一些属性
  • 默认属性
    1. title:     博文标题
    2. date:    博文创建时间
    3. tags:    博文的标签
  • 扩展属性
    1. update:     最后修改的时间
    2. comments:     是否允许评论此博文(默认true)
    3. categories:    博文种类
  • 配置文件(_config.yml)
              _config.yml 是专门用来配置Hexo功能的文件, 主要负责博客的框架(/Users/xxxx/hexo/hexo/_config.yml)

7.指令

1. init

/* 新建一个网站, 如果没有设置folder, Hexo默认在目前文件夹建立网站 */
hexo init [folder]

2. new

/* 新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。*/
hexo new [layout] <title>

3. generate

/*生成静态文件*/
/*选项            描述*/
/* -d, --deploy 文件生成后立即部署网站 */
/* -w, --watch  监视文件变动*/
hexo generate

4. publish

/*发布草稿*/
hexo publish [layout] <filename>

5. server

/*启动服务器, 默认情况下, 访问网址: http://localhost:4000/ */
/* 选项         描述*/
/* -p, --port     重设端口*/
/* -s, --static   只使用静态文件*/
/* -l, --log      启动日记记录, 使用覆盖记录格式*/
hexo server

6. deploy

/* 部署网站 */
/* 参数             描述*/
/* -g, --generate   部署之前预先生成静态文件*/
hexo deploy

7. render

// 渲染文件
// 参数          描述
// -o, --output     设置输出路径
hexo render <file1> [file2] ...

8. migrate

// 从其他博客系统 迁移内容。
hexo migrate <type>

9. clean

// 列出网站资料。
hexo list <type>

10.version

// 显示 Hexo 版本。
hexo version

11. 显示草稿

// 显示 source/_drafts 文件夹中的草稿文章。
hexo --draft

12. 一般你修改了东西的话只需要

hexo generate
hexo deploy
